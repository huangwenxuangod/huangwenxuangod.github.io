# 没有人把 Claude Design 当成一个剧本来看，但如果你这么看，你会发现现在只是第一幕结尾

**大多数人看到的是：Figma 跌了，AI 又赢了，设计行业要变天了。**

但我看到的是一个精心布局了两年的剧本，在4月17日这天，刚刚结束了它的第一幕。

---

## 序幕：4月17日发生了什么

4月17日，Anthropic 发布 Claude Design。

发布不到20分钟，Figma 股价开始下跌。当天收盘，Figma 跌了 6.84%。Adobe、Wix、GoDaddy 同日下跌。而这已经不是 Figma 今年第一次被 AI 打崩了——在这之前，它已经从 2025 年 8 月 IPO 高点跌掉了超过 85%。

很多人的反应是：哦，AI 又干掉一个行业了。

**但这个反应本身，就暴露了他们只看到了表面。**

我第一时间打开 Claude Design，用它做了一个金融 Dashboard 的原型——对标 RevenueCat 的设计风格，叫 FinanceCat。做完之后，我盯着屏幕看了好一会儿，没有说话。

不是因为震撼，是因为我突然意识到：这件事比我想的要大得多，但它现在还只是开始。

---

## 坐标系：这个剧本的三条主线

在拆之前，先说清楚我要从哪几个角度拆这件事。

**第一条主线：Anthropic 的战略意图。** Claude Design 不是一个设计工具，它是 Anthropic 打通整条产品链路的第一块拼图。

**第二条主线：Figma 的真实处境。** 营收涨 41%、股价跌 85%，这个矛盾背后是市场对一件事的判断——Figma 的商业模式，已经结构性失效了。

**第三条主线：普通人的机会窗口。** 每次技术范式切换，总有一批最早动手的人吃到最厚的红利。这次的窗口在哪，有多长。

这三条线串起来，才能看清楚这个剧本的完整结构。

---

## 幕后伏笔：Canva 铺了两年的局

所有人都以为 Claude Design 是 Anthropic 突然出手。

**其实这局从 2024 年就开始布了。**

2024 年，Canva 开始与 Anthropic 合作。2025 年 7 月，Canva 上线了 Canva MCP for Claude，让用户可以在 Claude 对话里直接生成 Canva 设计。到了 2026 年 1 月，集成扩展，支持品牌规范的自动应用。

然后是 4 月 17 日。Claude Design 正式发布，它的底层用的是 Canva 的 Design Engine。

这不是临时合作，这是两年的铺垫。**Canva 早就知道自己要成为 AI 设计的基础设施层，而不是跟 AI 竞争。** 它选择了做底层，让 Anthropic 做前台。

而同一天，Anthropic CPO Mike Krieger 悄悄辞去了 Figma 的董事会职务。

就在 Claude Design 发布前 3 天。

这种时机，不是巧合，是信号。当一家公司的 CPO 从潜在竞争对手的董事会辞职，意味着公司决定：**竞争比合作更重要。**

---

## 第一幕高潮：我做了一个金融 Dashboard，然后沉默了

我打开 Claude Design 做的第一个项目，是 FinanceCat——一个对标 RevenueCat 风格的个人财务 Dashboard。

我给了它一段描述，说我想要深色系、金融风格、数据密度高、有月度收支对比图。它生成了第一版。我看着这个界面，先是有点愣，然后开始往下翻。

**组件一致性，是我第一个注意到的东西。**

卡片的圆角、内边距、字重，从第一个 KPI 卡到最后一个图表，全部统一。这件事听起来很基础，但你真正用过 AI 设计工具就知道，这恰恰是最难做到的地方。Google Stitch 生成的东西，经常会在第3、第4个组件开始「漂移」——圆角突然变了，阴影深度不一致，整体看起来像是拼凑的。

Claude Design 没有这个问题。15 个组件，视觉语言完全统一。

**但我真正沉默的原因，不是好看。**

是当我打出第二句对话「把月度趋势图改成区域图，颜色用品牌橙」的时候，它改完之后，所有相关的组件配色也跟着联动了。

我没有告诉它要联动。它自己判断了这个改动的范围。

这件事如果发生在 Figma 里，我需要手动找到每一个用了这个颜色的地方，一个个去改。**而在 Claude Design 里，它理解的是「意图」，不是「操作指令」。**

这是本质上不同的东西。

---

## 它真正厉害的地方，不是「好看」

所有的媒体报道都在说 Claude Design 生成的设计有多精美。

**这是对了，但没说到点子上。**

Claude Design 真正改变的事情，是把「从想法到可交互原型」这条路上的摩擦力消失了。

以前这条路是什么样的？你有一个想法，然后你要么会 Figma、要么找设计师、要么画草图发需求文档、然后等待、然后对齐、然后改稿、然后再对齐。整个过程可能需要 3 天，快的话 1 天。

现在这条路是什么样的？你说出来，3 分钟以内，一个可以交互的原型出现在你眼前。

Brilliant（在线学习平台）在测试期间说了一句话：复杂的页面，其他工具需要 20 多个 prompt，Claude Design 只需要 2 个。

**不是提速 2 倍，是提速 10 倍。** 这不是同一个量级的改变。

当摩擦力消失，最先改变的不是工具本身，而是人们开始敢于「探索」这件事。以前你有 5 个方向，只敢做 1 个——因为做完一个已经耗完了时间。现在你可以把 5 个都做出来，然后比较。

这件事对产品决策的影响，比任何人现在说的都要深。

---

## Figma 的处境：营收涨 41%，股价跌 85%，这是矛盾吗

不是矛盾，这恰恰是最清晰的信号。

Figma 2025 年的营收是 10.6 亿美元，同比增长 41%，净收入留存率 136%——这意味着它的老客户不仅没有跑，还在付更多的钱。

但股价从 2025 年 8 月 IPO 高点跌了 85%。现在的价格，已经低于 IPO 发行价。

**市场在做的事情不是评估 Figma 今天的业绩，是在给它的商业模式的未来定价。**

这两件事完全可以同时成立：一家公司现在赚钱，同时它的核心商业逻辑在未来会失效。

Figma 赚钱靠的是什么？靠的是「专业设计工具的学习门槛」——你要做好设计，你就得学 Figma，你学了 Figma，你的团队就要买 Figma 的席位，你换工具的成本极高。这是它的护城河。

**而 Claude Design 干的事，是把这道门直接移走了。**

以前一个创业者要做产品原型，要么招设计师，要么自己学 Figma，至少需要几个月。现在他打开 Claude Design，今天就能出原型。

他不需要学 Figma 了。他甚至不需要知道 Figma 是什么。

---

## Figma 真正的护城河，AI 攻得进去吗

说清楚这件事很重要，因为很多人要么太悲观，要么太乐观。

**Figma 真正的护城河有两层。**

第一层是「存量」——全世界几万个团队已经在 Figma 里建好了他们的设计系统、组件库、历史稿件。这些东西迁移的成本极高，不是一朝一夕能动的。

第二层是「协作习惯」——Figma 是设计师、PM、工程师三方对齐的协作平台。这个三方协作的工作流，已经嵌进了无数团队的日常。

这两层护城河，Claude Design 现在都攻不进去。

**但 Claude Design 在做的事，是截断 Figma 的「入口」。**

以前学设计要经过 Figma，现在不用了。以前 PM 想出原型要找设计师用 Figma，现在 PM 自己就能做。以前创业者不得不学 Figma，现在有了更低成本的替代。

**Figma 的存量护城河还在，但增量正在被截断。** 这就是为什么营收还在涨，但市场判断它的未来已经失效。

---

## 第二幕预告：Design 和 Code 打通之后，会发生什么

这是我觉得最重要、也最少人在讲的部分。

现在的 Claude Design，和 Claude Code 之间的关系，是「可以 handoff」——你在 Design 里做好设计，打包成 handoff bundle，扔给 Claude Code，然后 Code 去实现。

这已经很强了。但这还不是第二幕。

**第二幕是 Design 和 Code 深度集成，中间的 handoff 消失，整条链路变成一个连续的 AI 流程。**

想象一下这个场景：你跟 Claude 描述你想要一个功能，它在设计层出原型，在代码层直接实现，在测试层自动验证，在部署层一键上线。

整条产品开发的链路，从「想法」到「上线」，全部在 Claude 体系里完成。

**链路上的每一个工具，都会面临被吞噬的压力。** Figma 是第一个被盯上的，因为它在链路最前端，且最容易被替代。但它不会是最后一个。

Datadog 的 PM 在测试期间说了一句话：「我们现在可以在会议结束前做出一个可以用的原型。」

**这句话的恐怖之处，不是「快」，是「在会议结束前」——也就是说，以前一个会议讨论出来的方向，还需要几天才能看到实物；现在，讨论和验证可以同时发生。**

决策的速度会变成一个完全不同量级的事情。

---

## 对开发者意味着什么

直说吧，前端开发者是这波冲击里最直接的承压者。

Claude Code 已经在改写「谁来写代码」这个问题的答案。现在加上 Claude Design，改写的是「谁来做产品」。

当一个 PM 可以自己出设计稿、自己用 Claude Code 生成前端代码、自己部署，他对「前端开发者」的需求是什么？

**需求没有消失，但性质变了。** 他不再需要一个「执行者」，他需要的是一个能处理 Claude 搞不定的边界问题的人。换句话说，**能跟 AI 协作的开发者，和不能的，价值差距会拉到极大。**

对于我这种学生来说，这件事的影响是直接的。我以前做项目，设计这一块是最头疼的——要么做出来很丑，要么找同学帮忙，要么砍掉视觉需求。

现在这个摩擦力消失了。我可以自己做出一个视觉质量合格的产品原型。这意味着我一个人能推进的项目复杂度，直接上了一个台阶。

**以前做一个完整的产品需要一个团队，现在一个人就够了。** 这不是在美化，这是我实测之后的判断。

---

## 对产品侧的影响：PM 的工作会变成什么样

这个问题我想了很久。

现在的 PM 工作流是：想法 → 写需求文档 → 找设计师出稿 → 对齐 → 找开发实现 → 对齐 → 上线。

中间有两个「找别人」的节点，这两个节点就是 PM 最大的时间黑洞——等待、对齐、修改、再对齐。

**Claude Design + Claude Code 的组合，把这两个节点都压缩了。**

PM 自己出设计稿，扔给 Claude Code 实现，整个验证周期从「天」变成「小时」。

这不意味着 PM 不再需要设计师和开发者。**但这意味着 PM 的工作重心会从「推进执行」变成「做判断」。** 那些花大量时间在对齐和传递上的低价值工作，会被 AI 吃掉。剩下的，是真正需要人来做的判断——用户洞察、策略选择、产品方向。

**PM 的门槛降低了，但优秀 PM 的稀缺性反而会提高。** 因为执行工作被 AI 拿走之后，暴露出来的，是你到底有没有真正的判断力。

---

## 第三幕的想象：AI 能不能自我迭代设计

这个部分我没有实据，只有推断。

但推断是有逻辑的。

当 Claude Design 可以读取用户行为数据——用户在哪个按钮停留最久、哪个页面跳出率最高、用户在哪里产生了犹豫——它能不能根据这些数据，自动提出设计修改方案？

技术上，这件事已经在发生的边缘了。

**第三幕不是「AI 设计了一个东西」，是「AI 在运行中持续优化一个东西」。** 设计不再是一个有终点的项目，而是一个持续运行的过程。

这件事成立的前提是：Claude Design 要深入产品的运行层，而不只是创作层。

这需要时间。但方向是清晰的。**当 Design 和 Code 完全打通，再加上数据层，AI 自我迭代设计这件事，是逻辑上的必然终点。**

---

## 这个剧本真正的反转是什么

我不知道。这是我的真实答案。

但我知道反转的方向在哪。

**当 Figma 这样的公司真正意识到「AI Native」是唯一的出路，并且做出真正的反击，那才是这个剧本最大的转折点。**

Figma 不是没有在动。它在 2026 年初推出了 Code to Canvas，让 AI 生成的代码可以直接转成 Figma 里的可编辑设计。这是一个防御动作，但它的逻辑方向还是「把 AI 的输出拉回 Figma」。

**这个方向是错的。** 真正的 AI Native 设计工具，不是把 AI 嵌进现有的工具流程，而是用 AI 重写整个工具的底层逻辑。

Figma 要做这件事，需要推翻自己。这是大公司最难做到的事。

反转什么时候发生，取决于 Figma 什么时候有这个勇气，或者什么时候被逼到不得不做这个选择。

---

## 普通人的机会窗口在哪

别看剧情分析了，说点现在能动手的。

**窗口期是真实存在的，但它很短。**

第一个机会：**用 Claude Design 给不懂设计的创业者和 PM 做原型服务。** 现在大多数人还不知道怎么用，信息差就是钱。一个完整的产品原型，收 2000-5000 块是合理的，用 Claude Design 做，你的时间成本可能只有 2-3 小时。

第二个机会：**做 Claude Design 的教程内容。** 这个工具太新了，真正深度测评的内容几乎没有。谁先做出来，谁先吃到流量红利。

第三个机会：**把它用进自己的产品开发流程。** 如果你在做独立开发或者副业项目，Claude Design + Claude Code 的组合，让你一个人能做的产品复杂度直接翻倍。

**这三个窗口，最多开 1-2 个月。** 过了这个时间点，教程内容烂大街，信息差消失，红利就没了。

---

## 你现在该站在哪个位置

这个问题比「用什么工具」更重要。

看这篇文章的人里，有设计师、有开发者、有 PM、有学生。**每个人在这个剧本里的处境不一样，该做的事也不一样。**

对设计师来说：**你的护城河永远不是「会用 Figma」，是「你有没有品味和判断力」。** AI 可以生成漂亮的东西，但它生成的东西对不对、好不好用、符不符合用户心智，这件事 AI 现在还给不出答案。

对开发者来说：**学会跟 AI 协作，比学任何具体技术都重要。** 会用 Claude Code 的开发者，和不会用的，正在成为两个物种。

对学生来说：**现在是最好的时机，因为你的沉没成本最低。** 没有工作流要维护，没有历史习惯要打破，你可以直接用 AI Native 的方式做事。

对 PM 来说：**用 Claude Design 亲手出一个设计稿，是现在最值得做的一件事。** 不是因为你要变成设计师，是因为「把想法变成实物」这个能力，是你理解用户和产品最快的路径。

---

## 第一幕的帷幕刚刚落下

**Figma 跌了 85%，不是这个故事的结局，是第一幕的高潮。**

Claude Design 发布，不是 AI 赢了设计行业，是第一幕最后一个镜头打出来了——所有的伏笔浮出水面，所有人终于意识到这个剧本一直在写。

第二幕是 Design 和 Code 打通，整条产品链路被 AI 接管。

第三幕是 AI 自我迭代，「设计」这件事的定义被重写。

真正的反转是什么，是 Figma 的反击，还是某个我们现在完全想不到的东西，我不知道。

**但我知道一件事：只有看清楚剧本结构的人，才能在对的时间出现在对的位置。**

大多数人现在在讨论「Claude Design 好不好用」。

而真正重要的问题是：**你打算在第二幕里扮演什么角色？**
