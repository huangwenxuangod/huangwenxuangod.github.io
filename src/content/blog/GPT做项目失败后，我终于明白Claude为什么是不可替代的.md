## 这不是偶然，是底层逻辑的区别

  

**那天的对比太强烈了，我开始思考：为什么两个模型的表现会差这么多？**

  

不是 GPT 能力不行，是它们的**底层逻辑不一样**。

  

GPT 想的是：「怎么让用户觉得我很厉害？怎么让用户开心？」

所以它先赞美你，给你看漂亮的架构图，说一堆你想听的话。至于最后能不能成，那是次要的。

  

Claude 想的是：「怎么帮用户把这件事做成？」

所以它先问清楚细节，补全你没说出来的信息，然后给你一个真正能落地的方案。

  

一个是「表演型」，一个是「做事型」。

  

这就是核心区别。

### 第一样：完整的 Python 脚本

```python

import os

import re

import smtplib

from datetime import datetime, timedelta

from email.mime.text import MIMEText

from email.mime.multipart import MIMEMultipart

  

# 配置

VAULT_PATH = "D:/Obsidian/Vault/每日笔记/"

EMAIL_FROM = "your-email@163.com"

EMAIL_TO = "your-email@gmail.com"

EMAIL_PASSWORD = "your-email-password"  # 163邮箱用授权码

  

def get_note_path():

    """获取今天的笔记文件路径"""

    today = datetime.now()

    filename = f"{today.year}年{today.month:02d}月{today.day:02d}日.md"

    return os.path.join(VAULT_PATH, filename)

  

def extract_content(note_path):

    """提取笔记中的特定内容"""

    if not os.path.exists(note_path):

        return "今日无记录"

  

    with open(note_path, "r", encoding="utf-8") as f:

        content = f.read()

  

    # 提取"今日完成"和"明日计划"部分

    result = []

    sections = ["今日完成", "明日计划"]

  

    for section in sections:

        # 匹配标题及其后的内容，直到下一个标题或文件结束

        pattern = rf"#+\s*{section}\s*\n(.*?)(?=\n#+|$)"

        match = re.search(pattern, content, re.DOTALL)

        if match:

            result.append(f"## {section}\n{match.group(1).strip()}")

  

    return "\n\n".join(result) if result else "今日无记录"

  

def send_email(subject, body):

    """发送邮件"""

    msg = MIMEMultipart()

    msg["From"] = EMAIL_FROM

    msg["To"] = EMAIL_TO

    msg["Subject"] = subject

  

    msg.attach(MIMEText(body, "plain", "utf-8"))

  

    # 163邮箱SMTP服务器

    with smtplib.SMTP_SSL("smtp.163.com", 465) as server:

        server.login(EMAIL_FROM, EMAIL_PASSWORD)

        server.send_message(msg)

  

def main():

    note_path = get_note_path()

    content = extract_content(note_path)

  

    today = datetime.now().strftime("%Y年%m月%d日")

    subject = f"日报 - {today}"

  

    send_email(subject, content)

    print(f"日报发送成功: {subject}")

  

if __name__ == "__main__":

    main()

```

  

这段代码是**真的能运行**的，每一个环节都写死了，没有 `pass`。

所以 GPT 会：

- 先赞美你，让你开心

- 给你看漂亮的架构图，让你觉得它很专业

- 写漂亮但不能用的代码，让你觉得它能力很强
